1. HTTP Status Code 有哪些？怎麼分類？

Ans:

HTTP 狀態碼的五大類
MDN Web Docs 
1xx (100–199) 資訊回應 (Informational)：表示請求已接收，正在處理中，這只是個臨時響應。
2xx (200–299) 成功回應 (Successful)：表示請求已成功接收、理解並處理完成。
3xx (300–399) 重新導向 (Redirection)：表示客戶端需要採取進一步行動以完成請求，通常是資源已移動到新位置。
4xx (400–499) 客戶端錯誤 (Client Error)：表示請求包含錯誤，例如語法錯誤、未授權或請求的資源不存在。
5xx (500–599) 伺服器錯誤 (Server Error)：表示伺服器在處理請求時發生了意外情況或無法完成請求。 
常見的 HTTP 狀態碼舉例
菜鳥教程 
200 OK：請求成功，一切正常。
301 Moved Permanently：資源永久移動到新 URL。
302 Found：資源臨時移動。
400 Bad Request：客戶端請求語法錯誤。
401 Unauthorized：需要身份驗證。
403 Forbidden：禁止訪問，權限不足。
404 Not Found：找不到請求的資源。
500 Internal Server Error：伺服器內部錯誤。
503 Service Unavailable：服務暫時不可用。
